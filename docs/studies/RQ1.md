# RQ1 — AI-Enhanced Coaching Efficacy

> Can a domain-trained LLM generate **valuable coaching insight** by
> integrating biomechanical (CV), kinematic (IMU), and physiological
> (HRV) data streams?

## Hypothesis

**H1.** Coaching advice generated from the **fused** payload
(`{CV, HRV, IMU}`) is rated **higher** by certified coaches than advice
generated from any single modality, on each of four criteria
(specificity, actionability, technical correctness, novelty).

**H0.** No mean-rank difference exists across the four payload modes.

## Design

Within-subjects, blinded rater study.

- **Conditions** (within-session): `cv`, `hrv`, `imu`, `fused`.
- **Stimulus**: corner advice generated by the same LLM
  (`google/gemma-4-e4b` via LM Studio) using the same system prompt
  (`coach.CORNER_ADVICE_SYSTEM_PROMPT`, version stored in
  `coach.PROMPT_VERSION`). The only thing that varies between conditions
  is the JSON payload subset shown to the model.
- **Cache**: every `(session_id, payload_mode, prompt_version)` tuple is
  generated **exactly once** and stored in `coach_advice_cache`. Every
  rater scores the same advice. Bumping `prompt_version` invalidates the
  cache.
- **Blinding**: the dashboard's `RQ1RaterCard` randomises the order of
  the four conditions per session using a deterministic seed
  (`mulberry32(hash(session_id))`). Labels are revealed only via a
  toggle.
- **Outcome variables**: 1–5 Likert scores for
  - specificity ("how concrete is the advice?")
  - actionability ("can the fighter act on this between rounds?")
  - correctness ("is the advice technically defensible?")
  - novelty ("does the advice surface something a coach watching the
    same round wouldn't have already said?")

## Inclusion criteria

A session is eligible iff **all** of:

1. `status = "completed"`
2. `round_count` set + `duration_ms > 0`
3. ≥ 1 punch event in `punch_event`
4. ≥ 1 HRV sample in `hr_sample`
5. ≥ 1 IMU sample in `imu_sample` (synthetic acceptable until the
   real Hykso/Corner trackers arrive)

The query that returns the eligible set lives at
`scripts/studies/rq1_sessions.py`.

## Sample size

Friedman test, k = 4 conditions, expected medium effect (Kendall's
W ≈ 0.20–0.30) → minimum detectable n ≈ 28 sessions at α = 0.05,
power = 0.80. Target **n = 30 sessions**, 3 raters per session.

## Raters

3+ certified boxing coaches (minimum credential: USA Boxing Level 1 or
international equivalent, ≥3 years coaching experience). Recruited
from local gyms. Each rater receives:

- A short orientation document (~1 page) describing the task and
  Likert anchors (e.g., specificity: 1 = vague generality, 5 =
  references concrete observations).
- Access to the dashboard with their assigned `rater_id`.

## Procedure

Per rater, per session:

1. Open `/sessions/{id}` on the dashboard.
2. Set the rater id (top of the RQ1 card) — persisted locally.
3. Click **Generate 4 variants**. The dashboard pulls the cached advice
   for each mode (`POST /sessions/{id}/advice?payload_mode=…`).
4. Read the four advice cards labelled A/B/C/D.
5. Score each card on all four criteria.
6. Optionally click **Reveal labels** to confirm assignment after
   scoring.

Ratings flow to the server via
`POST /studies/rq1/sessions/{id}/ratings`. Re-rating overwrites.

## Stopping rule

Stop when **n = 30 sessions × 3 raters = 90 rater-sessions × 4 modes ×
4 criteria = 1,440 ratings** are recorded. Run aggregation and stats.

If insufficient inter-rater reliability appears at n = 15 (ICC(3,k)
< 0.6), pause and revisit the rater orientation before continuing.

## Analysis

Pipeline:

1. `scripts/studies/rq1_aggregate.py` → `data/studies/rq1_ratings.csv`
2. `scripts/studies/rq1_stats.py` → `data/studies/rq1_stats.json`

Tests:

| Test | What it answers | Output |
|---|---|---|
| **Friedman χ²** per criterion | Is there an overall mode effect? | χ², p, n |
| **Friedman χ²** pooled | Mode effect across the criterion sum? | same |
| **Wilcoxon signed-rank** post-hoc | Which pairs of modes differ? | W, p, Bonferroni-corrected p |
| **Kendall's W** | How big is the mode effect? | 0–1 |
| **ICC(3,k)** per (mode, criterion) | Do raters agree? | 0–1 |

Significance threshold α = 0.05, two-sided.

## IRB

Study involves human raters scoring AI-generated text. No fighter PHI
is shown to raters beyond what already lives in the dashboard. IRB
exempt under DBA program guidelines for pedagogical / programmatic
evaluation studies. Confirm with GGU IRB before recruiting raters.

## Threats to validity

- **Generation variance**: gemma-4-e4b is non-deterministic at
  temperature 0.5. Cache solves this for the actual study, but the
  cached generation is *one sample* from the model's distribution.
  Document the model + temperature in the chapter.
- **Order effects**: mitigated by the per-session shuffle. Raters who
  rate many sessions will see modes in different positions each time.
- **Coach pool homogeneity**: all from the local boxing community;
  results may not generalise to MMA / kickboxing coaches.
- **Synthetic IMU**: until the Polar H10 + wrist sensors land, IMU is
  synthesised from CV punches. The fused condition therefore receives
  some modality redundancy. Note this caveat in the writeup.
